User Documentation

Installation:

 

Step 1: start sqlplus load files:

Setup.sql

sequence.sql

index.sql

job.sql 3 times with the following parameters

1.    subindex 0.5

2.    plaindex 0.5

3.    desindex 0.5

 

Step 2: start tomcat by typing 'starttomcat' in terminal

Step 3: Use the following address in your browser to access the HTML file:http://”name of target machine”.cs.ualberta.ca:"the-first-port- number"/project/ login.html

 

 

Login:

1)        Either your user name or password is not supposed to longer than 24 characters, then click the Login button.

2)        If do not have an account, the user could register by clicking the ‘register’ button and entering the indispensable information.

3)        If the user wants to log out, there is a button named ‘Log Out’ on the main page.

 

 

User Management:

After login, the user will come to the main page, and his information will be posted on the top of page.

More specifically, the user may use this page to perform the following tasks:

 

1)            Change the password:

There is a button named ‘Change Password’ on the main page. The user could click that button to skip to the changing page. There only one box for entering a new password, and do not forget to click ‘change’ button to go back the main page.

 

2)            Update user information:

There is a button named ‘Update Profile’ on the main page. The user could click that button to skip to the update page. The user’s information still is posted on the top of page, and there are corresponding empty boxes for entering new information. After the user submit the new information, the data will be updated immediately.

 

3)            See top 5 pictures

There is a button named ‘See Top 5 Pictures!’ on the main page. The user could click that button to skip to the new page. That page will display the top five most popular images. A popularity of an image is specified by the number of distinct users that have ever viewed the image (not its thumbnail). In case of a tie, display all tired images.

 

4)            Add a new group:

There is a button named ‘Add Groups!’ on the main page. The user could click that button to skip to the group page. The user could enter the name of group and the names who could share pictures in the group.

5)            Manage the groups:

If user want change their group, there is a button named ‘Manage your groups!’ for changing information. On that page, the user should choose group name first. Then another page will be posted, the user could delete or add the friends who could share pictures in this group.

 

6)            Upload a new picture:

There is a button named ‘Upload Pictures!’ on the main page. The user could click that button to skip to the upload page. On the upload page, the user should select one or more picture from a local directory and upload it. Then another page will be post, the user could enter all information to that picture.

 

 

7)             Data analysis:

This function is only for admin account user. There is a button named ‘Data analyse!’ on the main page. The user could click that button to skip to the data analysis page. On that page, there are six options to group by: ‘user’, ‘subject’, ‘weekly’, ‘monthly’, ‘yearly’ and ‘all’. ‘Weekly’, ‘monthly’, ‘yearly’ and ‘all’ only could be chose one of them.

The user also should enter a keyword for search.

 

8)            Query the database to see relevant items

The user could enter a keyword that he wants to search, a start date and an ending date after the corresponding headings.  If the keywords are empty, then to search all picture this user could access. And there are there sorting orders: most-recent-first, most-recent-last and relevant.

9)            Details of pictures

For the result of search, the user could click the thumbnail to show the original size picture.

The page with the original size picture will post the information about picture together.

And there are two buttons on that page: update and remove, but they only could be used by the owner.
